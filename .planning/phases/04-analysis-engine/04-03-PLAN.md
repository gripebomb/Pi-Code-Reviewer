  <name>Task 2: Create validation script for path references and report structure</name>
  <files>test/validate-analysis.js</files>
  <action>
Create `test/validate-analysis.js` that validates generated review reports against requirements.

The script should:

1. **Parse REVIEW.md**:
   - Extract all findings (lines starting with `- **`)
   - Extract category sections (## Code Quality, etc.)
   - Extract severity sub-sections (### High, ### Medium, ### Low)

2. **Validate Structure**:
   - All 5 categories exist (Code Quality, Refactoring, Documentation, Security, Test Coverage)
   - Each category has a summary assessment (1-2 sentences before any findings)
   - Severity sub-sections are omitted only when empty

3. **Validate Path References**:
   - Extract all backtick-quoted paths from findings
   - Verify each path exists in the fixture repository
   - Count findings with vs. without path references
   - Report paths that don't exist

4. **Validate Severity**:
   - Count findings per severity level per category
   - Flag categories with only High findings (possible severity inflation)
   - Flag categories with no findings but missing summary

5. **Output Results**:
   - Print validation summary with pass/fail per check
   - List any invalid path references
   - Show finding counts per category and severity
   - Exit with code 0 if all checks pass, 1 if any fail

Usage:
```bash
node test/validate-analysis.js test/fixtures/fixture-a/.planning/REVIEW.md test/fixtures/fixture-a/
```

The script should be robust against markdown formatting variations and handle edge cases (empty categories, no findings, etc.).
