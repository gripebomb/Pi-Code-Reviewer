Create a CLI wrapper and update package.json for CLI support (per D-08, D-09).

1. **Create `bin/pi-code-reviewer.js`**:
   ```javascript
   #!/usr/bin/env node

   /**
    * pi-code-reviewer CLI
    *
    * Usage:
    *   pi-code-reviewer [options]
    *
    * This is a thin wrapper that prints instructions for using the skill
    * inside Pi. The actual review logic lives in the skill instructions.
    */

   const args = process.argv.slice(2);

   if (args.includes('--help') || args.includes('-h')) {
     console.log(`
   pi-code-reviewer — Code review skill for Pi

   Usage:
     pi-code-reviewer [options]

   Options:
     --help, -h     Show this help message
     --version, -v  Show version number

   This package is designed to be used as a Pi skill.
   Install it with:

     pi install pi-code-reviewer

   Then invoke it inside Pi with:

     /skill:code-reviewer

   The skill will analyze your repository and produce:
     - .planning/REVIEW.md — comprehensive review report
     - .planning/REVIEW-TODO.md — actionable checklist
   `);
     process.exit(0);
   }

   if (args.includes('--version') || args.includes('-v')) {
     const pkg = require('../package.json');
     console.log(pkg.version);
     process.exit(0);
   }

   console.log(`
   pi-code-reviewer — Code review skill for Pi

   This package is designed to be used as a Pi skill.
   Install it with:

     pi install pi-code-reviewer

   Then invoke it inside Pi with:

     /skill:code-reviewer

   For more information:
     pi-code-reviewer --help
   `);
   process.exit(0);
   ```

   Make the file executable: `chmod +x bin/pi-code-reviewer.js`

2. **Update `package.json`**:
   Add to the root of package.json:
   ```json
   "bin": {
     "pi-code-reviewer": "./bin/pi-code-reviewer.js"
   }
   ```

   Add to the `files` array:
   ```json
   "files": ["README.md", "skills/", "bin/"]
   ```

   Ensure the JSON remains valid and all existing fields are preserved.

3. **Create `.npmignore`** (if it doesn't exist):
   ```
   test/
   scripts/
   .planning/
   .git/
   .gitignore
   ```
   This ensures test fixtures, validation scripts, and planning artifacts are not included in the npm package.

1. **Pre-publish validation**:
   - Run `scripts/validate-phase-05.mjs` (release-readiness checks)
   - If validation fails, abort with error message
   - Verify git working directory is clean (no uncommitted changes)

2. **Version check**:
   - Read current version from package.json
   - Verify version is not already published (optional — check npm view)
   - Print the version that will be published

3. **Pack test**:
   - Run `npm pack --dry-run` to verify tarball contents
   - Verify tarball includes: README.md, skills/, bin/
   - Verify tarball does NOT include: test/, scripts/, .planning/

4. **Publish**:
   - Run `npm publish` (or `npm publish --access public` if scoped package)
   - Capture publish output

5. **Post-publish verification** (per D-10, D-11):
   - Create a temporary directory
   - Run `npm install pi-code-reviewer@latest` in the temp directory
   - Verify the package installs without errors
   - Verify `node_modules/pi-code-reviewer/skills/code-reviewer/SKILL.md` exists
   - Verify `node_modules/pi-code-reviewer/bin/pi-code-reviewer.js` exists
   - Run `npx pi-code-reviewer --version` and verify it outputs the version
   - Verify the skill is discoverable by checking the pi.skills field in the installed package.json

6. **Cleanup**:
   - Remove temporary directory
   - Print summary of publish and verification results

7. **Output format**:
   ```javascript
   console.log('Publish and Verify Pipeline');
   console.log('==========================');
   // ... per-step output
   
   if (allPassed) {
     console.log('\n✓ Package published and verified successfully');
     console.log(`  Version: ${version}`);
     console.log(`  Registry: https://www.npmjs.com/package/pi-code-reviewer`);
     process.exit(0);
   } else {
     console.log(`\n✗ Pipeline failed at step: ${failedStep}`);
     process.exit(1);
   }
   ```

Use ES module syntax. Use `child_process` for running npm commands. Follow patterns from existing scripts.
